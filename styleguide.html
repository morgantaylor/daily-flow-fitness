Introduction

As Buffer continues to grow, and the number of people needing static pages increases, it will be important for us to maintain consistent styles and structures. This is an early version of a "static page style guide" that can be referenced while constructing a page for any need.

Hero

Each static page can have one primary hero container. You can see this page's hero at the very top (scroll up!). Hero sections can have solid background colors or use images as the background. Images should be large, and wider than they are tall (to avoid distortion).

Example: Hero <section class="hero <options>" id="js-hero" style="<style-options>">
  <div class="content">
    <div class="container">
      <h1 class="hero-heading">Hero Main Heading</h1>
      <h2 class="hero-subheading">Hero Subheading Goes Here</h2>
      <a class="button btn-outline hero-cta">Optional Primary CTA</a>
    </div>
  </div>
</section>

Options:
  • light-background
  • dark-background
  • hero-splash

    • The hero-splash modifier will make the hero image fill up the entire page. Toggle the splash page setting at the top of this page to see an example.


Style Options:
  • background-color: #color;
  • background-image: #fallback-color url('static-image-url') no-repeat;

Replace the <options> as necessary, depending on the background being used. The <style-options> allow you to use a custom color or image as the hero background.

Note: You can also use a primary button cta, as documented above, if there is one primary action for a person to take.

Starting a Page

After each page hero, you will need to start each page with two lines of code:

Example: Starting a Page <div class="page-content">
  <div class="page-layout <sidebar-options>">

Sidebar Options:
  • with-sidebar
  • without-sidebar

If the page will not have a sidebar, use without-sidebar and add this extra line of code below the previous two lines:

Example: Without a Sidebar <div class="page-content-container" id="js-page-content-container">

If the page will have a sidebar, see the step: Sidebar

Sidebar

If you want to have a sidebar enabled on the page, make sure that the sidebar-options from the previous step is set to with-sidebar

Next, add the following code after the page-layout div:

Example: Sidebar <div class="sidebar-container" id="js-sidebar-container">
  <section class="sidebar" id="js-sidebar">
    <ul class="sidebar-list">
      <a href="#" class="sidebar-list-item"><li>Example List Item</li></a>
    </ul>

    <ul class="sidebar-list">
      <span class="sidebar-list-title">Example List Title</span>
      <a href="#" class="sidebar-list-item"><li>Example List Item</li></a>
    </ul>
  </section>
</div>

<div class="page-content-container" id="js-page-content-container">

You can modify the above code to create a single list or sub-lists.

Paragraphs

We try to optimize clarity and legibility with longer blocks of text. Here's how to insert a paragraph onto a page:

Example: Divider <section class="paragraph">
  <div class="container">
    <h4 class="section-title" id="introduction">Optional title for a section</h4>
    <p>Your paragraph text goes here.</p>
    <p>And will work with multiple paragraphs.</p>
  </div>
</section>

Grids

Grids are great for showing multiple pieces of related content in the same section. Grids can be broken down into fifths, fourths, thirds and halves. Here's a quick demo (see the style guide for further reference):

Example: Grids <section class="grid <grid-options>">
  <div class="container">
    <div class="grid-row">
      <div class="col one-fourth">Content here</div>
      <div class="col one-fourth">Content here</div>
      <div class="col one-fourth">Content here</div>
      <div class="col one-fourth">Content here</div>
    </div>
  </div>
</section>

Grid Options:

  • blank (default)
  • full-width

A full-width grid will break its container to fill the entire width of the browser window. A clear example of a full-width grid is the Transparency Dashboard.

Here are the semantics for different grid sizes:

  • class="col one-half"
  • class="col one-third"
  • class="col two-third"
  • class="col one-fourth"
  • class="col one-half"
  • class="col three-fourth"
  • class="col one-fifth"
  • class="col two-fifth"
  • class="col three-fifth"

Note: create a new grid-row div to add a new layer to the page. The contents of each grid-row should add up to one whole (i.e. one-fourth + one-fourth + one-half = one).

Additionally, the col class in each grid item stands for column. It is needed on each grid element.

Calls to Action

You can create responsive, full-width calls to action with the cta component. You will need to set either a dark-background or light-background class depending on the background image you wish to use.

See the style guide for further reference on button types that can be used here!

Example: Call to Action <section class="call-to-action dark-background" style="background: url('/images/about/about-hero.jpg') no-repeat; background-size: cover;">
  <div class="container">
    <h2 class="cta-heading">Call to action headline</h2>
    <h3 class="cta-subheading">Call to action subheading</h3>
    <a href="#" class="large button btn-outline cta-action">Primary Action</a>
  </div>
</section>

Ending a Page

At the very end of each page, we need to close the three high-level containing elements. To do this, simple add the following code at the end of the document:

Example: Ending a Page       </div><!-- page container -->
    </div><!-- page layout -->
  </div><!-- page content -->
